ArrayLists

Arrays list implements a list using an array and can only hold objects NOT elements of a primitive data type. Unlike arrays, it allows for dynamic resizing of the underlying array, making it a flexible data structure for storing and manipulating data.

Here is an example of creating an ArrayList using the 'ArrayList' class by specifying the object type it holds within the angle brackets.

JAVA
  ArrayList<String> fruits = new ArrayList<>();

You can access elements in an ArrayList using the get() method, which takes the index of the element you want to retrieve. Remember that the index starts from 0. Throws IndexOutOfBoundsException when the index is less than size or greater or equal to the ArrayList size. You can update an element in an ArrayList by using the set() method, which takes the index of the element to be updated and the new value.

JAVA
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Orange");
String firstFruit = fruits.get(0);
System.out.println("First fruit: " + firstFruit); // First fruit: Apple
fruits.set(1, "Mango");
String firstFruit = fruits.get(0);
System.out.println("First fruit: " + firstFruit); First fruit: Mango

Iteration

There are two common ways to iterate over an ArrayList in Java, either using a traditional for loop or using a for-each loop. Let's explain each method and provide coding examples for better understanding:

The traditional for loop provides more control and flexibility during iteration. It allows us to explicitly manage the index and access elements using the get() method. The for loop, will loop through all the indexes from 0-size()-1 and access the element using the get method. Here's an example

JAVA
for (int i = 0; i < fruits.size(); i++) {
    String fruit = fruits.get(i);
    System.out.println(fruit);
}

The for-each loop is a simpler and more concise way to iterate over an ArrayList (Or any other iteratable objects). It automatically takes care of the iteration and provides access to each element without needing to manage the index explicitly. This loop is especially useful when you want to perform a specific action on each element of the ArrayList, without requiring the index. The loop uses a variable declared inside the loop statement to loop through the entire ArrayList.

JAVA
  for (String fruit : fruits) {
      // Do something with each fruit
      System.out.println(fruit);
  }

Autoboxing & Autounboxing

Autoboxing and Autounboxing are very important while using ArrayLists as they are automatic conversions between primitive types and their corresponding wrapper classes. This is important to understand when working with ArrayLists because ArrayLists can only store objects (reference types) and not primitive types directly. Autoboxing is the proces converting a primitive data type to its corresponding wrapper class for example an int to an Integer object. When you add a primitive value to an ArrayList, autoboxing automatically converts the primitive value to its wrapper class object allowing the code to work. Autounboxing is when converting a wrapper class into its corresponding primitive data type. When you retrieve a value from an ArrayList that contains wrapper class objects, unboxing automatically converts the wrapper class object to its primitive type.

Useful methods

  • set(index, element): Replaces the element at the specified index with a new element.
  • remove(index): Removes the element at the specified index from the ArrayList.
  • size(): Returns the number of elements in the ArrayList.
  • isEmpty(): Checks if the ArrayList is empty.
  • contains(element): Checks if the ArrayList contains a specific element.
  • clear(): Removes all elements from the ArrayList.
  • indexOf(element): Returns the index of the first occurrence of the specified element, -1 if it does not.
  • toArray(): Converts the ArrayList to an array.
